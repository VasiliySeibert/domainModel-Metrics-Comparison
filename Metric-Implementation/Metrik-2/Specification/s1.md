# S1 Specification – Graph Edit Distance for Class Models

```
metric(instructor_model, student_model):
    instructor_graph = createGraph(instructor_model)
    student_graph    = createGraph(student_model)
    element_mapping  = computeElementMapping(instructor_graph, student_graph)
    relation_mapping = computeRelationMapping(instructor_graph, student_graph)
    optimal_mapping  = computeOptimalMapping(instructor_graph, student_graph, element_mapping, relation_mapping)
    vertex_edit_informations = extractVertexEditInformations(optimal_mapping)
    edge_edit_informations   = extractEdgeEditInformations(optimal_mapping)
    edit_informations = aggregateEditInformations(vertex_edit_informations, edge_edit_informations)
    edit_informations = scaleEditCosts(edit_informations, instructor_graph, student_graph)
    return edit_informations

    requires:
        isValidModel(instructor_model),
        isValidModel(student_model)
    ensures:
        isValidEditInformations(edit_informations)
```

## Invariants

```
isValidModel(M):
    checks if model adheres to the data structure.
    class names are unique within M
    enumeration names are unique within M
    every relation references classes that exist in M
    every attribute belongs to exactly one class in M
```

```
isValidGraph(G):
    G is an attributed undirected multigraph G = (V, E, μ, ρ)
    V is a finite set of vertices; E ⊆ V × V is a finite set of edges
    μ: V → M is a bijection mapping each vertex to a unique model element
    ρ: E → Z is a bijection mapping each edge to a unique subset of relations
    Each element m ∈ M has form (ν_m, φ_m, k_m, A, O) with:
        ν_m ∈ L_M   (unique element name)
        φ_m ∈ S_M   (scope variant)
        k_m ∈ {class, interface}
        A   (set of attributes)
        O   (set of operations)
    Each attribute a ∈ A has form (ν_a, τ_a, φ_a) with:
        ν_a ∈ L_A   (attribute name)
        τ_a ∈ T_A   (data type)
        φ_a ∈ S_A   (scope variant)
    Each operation o ∈ O has form (ν_o, τ_o, φ_o, P) with:
        ν_o ∈ L_O   (operation name)
        τ_o ∈ T_O   (return type)
        φ_o ∈ S_O   (scope variant)
        P   (set of parameters)
    Each parameter p ∈ P has form (ν_p, τ_p) with:
        ν_p ∈ L_P   (parameter name)
        τ_p ∈ T_P   (parameter type)
    Each relation r ∈ R has form (k_r, b_s, b_t) with:
        k_r ∈ T_R   (relation kind)
        b_s, b_t are valid relation ends
    Element names are unique within M
    Every relation references elements that exist in M
    Every attribute and operation belongs to exactly one element in M
```

```
isValidMapping(mapping):
    mapping is a Mapping instance with the following structure:
      element_cost_matrix   : dict[(instructor_vertex_id, student_vertex_id), float]
      relation_cost_matrix  : dict[(instructor_edge_id, student_edge_id), float]
      vertex_mappings       : list of VertexMappingEntry
      edge_mappings         : list of EdgeMappingEntry
      unmapped_instructor_vertices : list of vertex ids
      unmapped_student_vertices    : list of vertex ids
      unmapped_instructor_edges    : list of edge ids
      unmapped_student_edges       : list of edge ids
      total_raw_cost        : float

    1. mapping is a Mapping instance
    2. All vertex mapping entries are VertexMappingEntry instances
    3. All edge mapping entries are EdgeMappingEntry instances
    4. No instructor vertex mapped twice
    5. No student vertex mapped twice
    6. No instructor edge mapped twice
    7. No student edge mapped twice
    8. No overlap between mapped and unmapped instructor vertices
    9. No overlap between mapped and unmapped student vertices
    10. No overlap between mapped and unmapped instructor edges
    11. No overlap between mapped and unmapped student edges
    12. All raw costs in vertex mappings are in [0, 1]
    13. All raw costs in edge mappings are in [0, 1]
    14. No negative costs in cost matrices
    15. total_raw_cost is non-negative
```

```
isValidEditInformations(edit_informations):
    edit_informations is an EditInformations instance with the following structure:
      operations              : list of EditInformation
      total_scaled_distance   : float

    1. edit_informations is an EditInformations instance
    2. operations is a list
    3. Every item is an EditInformation instance
    4. Every operation_type is in {vertex_substitution, vertex_deletion, vertex_insertion,
                                   edge_substitution, edge_deletion, edge_insertion}
    5. source_ref is either a valid vertex/edge id from instructor_graph or None (ε)
    6. target_ref is either a valid vertex/edge id from student_graph or None (ε)
    7. raw_cost is numeric, raw_cost ≥ 0
    8. raw_cost = 1 for all insertion and deletion operations
    9. raw_cost ∈ [0, 1] for all substitution operations
    10. scaled_cost is numeric and scaled_cost ∈ [0, 1]
    11. total_scaled_distance is numeric and total_scaled_distance ∈ [0, 1]
    12. No duplicate edit operations (same type, source_ref, target_ref) in the list
    13. The operations collectively form a complete edit path transforming all vertices
        and edges of instructor_graph into student_graph
```

## Decompose functions

```
graph = createGraph(model):
    V = createVertexForEachElement(model.classes, model.interfaces)
    E = createEdgeForEachRelation(model.relations)
    μ = bijective map from V to model elements
    ρ = bijective map from E to subsets of relations between elements
    return G = (V, E, μ, ρ)

    requires:
        isValidModel(model)
    ensures:
        isValidGraph(G)

Input:  A parsed PlantUML model adhering to isValidModel.
Output: A graph instance adhering to isValidGraph.
```

```
mapping = computeElementMapping(instructor_graph, student_graph):
    for each pair of elements (m, m') where m ∈ instructor_graph, m' ∈ student_graph:
        compute intra-level tree distance δ(m, m') using recursive vertex matching
        - structure vertices: ordered matching of name, kind, scope, attributes-set,
          operations-set using weighted sum δ_str with weights summing to 1
        - set vertices: unordered matching of attributes, operations, parameters using
          Hungarian algorithm on extended sets with ε-padding; cost of ε-pairing is 1
        - leaf vertices: atomic distance for names (Hamming or edit distance normalised),
          data types (compatibility table), scopes (Table 2), etc.
    store all δ(m, m') in mapping.element_cost_matrix
    return mapping

    requires:
        isValidGraph(instructor_graph),
        isValidGraph(student_graph)
    ensures:
        isValidMapping(mapping),
        all values in mapping.element_cost_matrix are in [0, 1],
        diagonal entries (m, m) are 0,
        matrix is symmetric when models are swapped

Input:  Two valid graph instances representing instructor and student class models.
Output: A mapping instance with element_cost_matrix populated.
```

```
mapping = computeRelationMapping(instructor_graph, student_graph):
    for each pair of relation sets (R_mn, R_m'n') assigned to edges (e, e'):
        compute intra-level tree distance δ(R_mn, R_m'n') using recursive vertex matching
        - root vertex is the relation set; children are individual relations
        - structure vertices: ordered matching of relation kind, source-end set, target-end set
        - set vertices: unordered matching of relation ends using Hungarian algorithm;
          source and target ends are treated as a set B because associations are reciprocal
        - leaf vertices: atomic distance for relation kinds (Table 3), role names,
          multiplicities (Eq. 15), and navigability {0, 1}
    store all δ(R_mn, R_m'n') in mapping.relation_cost_matrix
    return mapping

    requires:
        isValidGraph(instructor_graph),
        isValidGraph(student_graph)
    ensures:
        isValidMapping(mapping),
        all values in mapping.relation_cost_matrix are in [0, 1],
        diagonal entries are 0,
        matrix is symmetric when models are swapped

Input:  Two valid graph instances.
Output: A mapping instance with relation_cost_matrix populated.
```

```
mapping = computeOptimalMapping(instructor_graph, student_graph, element_mapping, relation_mapping):
    initialise OPEN with empty edit path
    while OPEN not empty:
        remove partial path λ with minimum estimated cost c(λ) = g(λ) + h(λ)
        if λ is complete:
            break
        expand λ with all applicable operations for next unmapped vertex:
            - vertex substitution (u → u') using element_mapping.element_cost_matrix
            - vertex deletion    (u → ε)  with cost 1
            - vertex insertion   (ε → u') with cost 1
        for each vertex operation, apply implied edge operations:
            - edge substitution ((u,v) → (u',v')) using relation_mapping.relation_cost_matrix
            - edge deletion     ((u,v) → ε)       with cost 1
            - edge insertion    (ε → (u',v'))     with cost 1
        prune paths using beam search (keep only most promising paths by c(λ))
    store optimal vertex/edge mappings, unmapped items, and total_raw_cost in mapping
    return mapping

    requires:
        isValidGraph(instructor_graph),
        isValidGraph(student_graph),
        isValidMapping(element_mapping),
        isValidMapping(relation_mapping)
    ensures:
        isValidMapping(mapping),
        mapping.vertex_mappings and mapping.edge_mappings form a bijective optimal mapping

Input:  Two valid graphs and two valid mappings with populated cost matrices.
Output: A mapping instance with optimal vertex/edge mappings and total_raw_cost.
```

```
edit_informations = extractVertexEditInformations(mapping):
    for each vertex_mapping_entry in mapping.vertex_mappings:
        create EditInformation with:
            operation_type = vertex_substitution
            source_ref = instructor_vertex_id
            target_ref = student_vertex_id
            raw_cost   = raw_cost from entry
    for each unmapped_instructor_vertex in mapping.unmapped_instructor_vertices:
        create EditInformation with:
            operation_type = vertex_deletion
            source_ref = instructor_vertex_id
            target_ref = None   (ε)
            raw_cost   = 1
    for each unmapped_student_vertex in mapping.unmapped_student_vertices:
        create EditInformation with:
            operation_type = vertex_insertion
            source_ref = None   (ε)
            target_ref = student_vertex_id
            raw_cost   = 1
    return edit_informations

    requires:
        isValidMapping(mapping)
    ensures:
        isValidEditInformations(edit_informations)

Input:  A valid mapping instance with optimal vertex mappings and unmapped vertices.
Output: An edit_informations instance containing all vertex-level operations.
```

```
edit_informations = aggregateEditInformations(vertex_edit_informations, edge_edit_informations):
    combined_operations = vertex_edit_informations.operations + edge_edit_informations.operations
    sort combined_operations by operation order implied by the optimal mapping
    total_raw = sum(op.raw_cost for op in combined_operations)
    return EditInformations(operations=combined_operations, total_scaled_distance=0.0)

    requires:
        isValidEditInformations(vertex_edit_informations),
        isValidEditInformations(edge_edit_informations)
    ensures:
        isValidEditInformations(edit_informations)

Input:  Two valid edit_informations instances (vertex and edge operations).
Output: A single combined and ordered edit_informations instance.
```

```
edit_informations = scaleEditCosts(edit_informations, instructor_graph, student_graph):
    c_max = max(|V_instructor|, |V_student|) + |E_instructor| + |E_student|
    for each op in edit_informations.operations:
        op.scaled_cost = op.raw_cost / c_max
    total_raw = sum(op.raw_cost for op in edit_informations.operations)
    edit_informations.total_scaled_distance = total_raw / c_max
    return edit_informations

    requires:
        isValidEditInformations(edit_informations),
        isValidGraph(instructor_graph),
        isValidGraph(student_graph)
    ensures:
        isValidEditInformations(edit_informations),
        edit_informations.total_scaled_distance ∈ [0, 1]

Input:  A valid edit_informations instance and the two original graphs.
Output: The same edit_informations instance with scaled_cost fields populated
        and total_scaled_distance computed.
```
