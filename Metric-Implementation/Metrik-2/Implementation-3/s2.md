# S2 Specification – Normalized Multi-Dimensional Metric

Extends S1 by decomposing the aggregate graph-edit distance into three human-evaluation-aligned scores.

## Overview

S1 produces a single scalar `total_scaled_distance` ∈ [0, 1].
S2 normalizes the raw edit operations into a **MetricResult** dict with three keys:
- `class_score`       — similarity in detected classes
- `attribute_score`   — similarity in attribute sets
- `association_score` — similarity in relationships

These scores are aligned with the human evaluation F1 categories used in `metric_interface.py`.

## Pipeline

```
metric(instructor_model, student_model):
    # ---- S1 pipeline (unchanged) ----
    instructor_graph = createGraph(instructor_model)
    student_graph    = createGraph(student_model)
    element_mapping  = computeElementMapping(instructor_graph, student_graph)
    relation_mapping = computeRelationMapping(instructor_graph, student_graph)
    optimal_mapping  = computeOptimalMapping(
        instructor_graph, student_graph, element_mapping, relation_mapping
    )
    vertex_edit_informations = extractVertexEditInformations(optimal_mapping)
    edge_edit_informations   = extractEdgeEditInformations(optimal_mapping)
    edit_informations = aggregateEditInformations(
        vertex_edit_informations, edge_edit_informations
    )
    edit_informations = scaleEditCosts(
        edit_informations, instructor_graph, student_graph
    )

    # ---- S2 decomposition ----
    result = normalization(
        edit_informations,
        optimal_mapping,
        element_mapping,
        instructor_graph,
        student_graph,
    )

    return result

    requires:
        isValidModel(instructor_model),
        isValidModel(student_model)
    ensures:
        validate_metric_result(result) == True
        result.class_score       ∈ [0, 1]
        result.attribute_score   ∈ [0, 1]
        result.association_score ∈ [0, 1]
```

## Decomposition Function

```
result = normalization(edit_informations, optimal_mapping, element_mapping,
                       instructor_graph, student_graph):

    # ---- class_score ----
    # Based on vertex-level edit operations (VERTEX_*).
    # The maximum possible class cost is max(|V_inst|, |V_stud|).
    vertex_raw_costs = sum(op.raw_cost for op in edit_informations.operations
                           if op.operation_type starts with VERTEX)
    class_score = 1.0 - (vertex_raw_costs /
                         max(|V_inst|, |V_stud|))
    class_score = clamp(class_score, 0.0, 1.0)

    # ---- association_score ----
    # Based on edge-level edit operations (EDGE_*).
    # The maximum possible association cost is |E_inst| + |E_stud|.
    edge_raw_costs = sum(op.raw_cost for op in edit_informations.operations
                         if op.operation_type starts with EDGE)
    association_score = 1.0 - (edge_raw_costs /
                               (|E_inst| + |E_stud|))
    association_score = clamp(association_score, 0.0, 1.0)

    # ---- attribute_score ----
    # Attribute-only distance for every mapped vertex pair.
    # For each VertexMappingEntry in optimal_mapping.vertex_mappings:
    #     look up the instructor element   m  via instructor_graph μ
    #     look up the student    element   m' via student_graph    μ
    #     compute δ_attrs(m, m') = Hungarian distance between attribute sets
    #                              with weights: name=0.5, type=0.3,
    #                              const=0.1, default=0.1
    #     if m and m' are different classes but have 0 attributes each:
    #         δ_attrs = 0.0
    #     normalise by max(|attrs(m)|, |attrs(m')|)
    #
    # For unmapped (deleted/inserted) vertices:
    #     count as max penalty = 1.0 per vertex
    #
    # attribute_penalty = (sum of δ_attrs for mapped pairs
    #                     + 1.0 * #unmapped_instructor_vertices
    #                     + 1.0 * #unmapped_student_vertices)
    # max_attr_cost = max(|V_inst|, |V_stud|)   (worst case: every class has
    #                                            max substitution cost 1)
    # attribute_score = 1.0 - (attribute_penalty / max_attr_cost)
    # attribute_score = clamp(attribute_score, 0.0, 1.0)

    return MetricResult(
        class_score=class_score,
        attribute_score=attribute_score,
        association_score=association_score,
    )

    requires:
        isValidEditInformations(edit_informations),
        isValidMapping(optimal_mapping),
        isValidMapping(element_mapping),
        isValidGraph(instructor_graph),
        isValidGraph(student_graph)
    ensures:
        validate_metric_result(result) == True
        result.class_score       ∈ [0, 1]
        result.attribute_score   ∈ [0, 1]
        result.association_score ∈ [0, 1]
```

## Return Type

The return value must be a plain `dict` with exactly these keys and value constraints:

```python
{
    "class_score":       float,  # 0.0 ≤ x ≤ 1.0
    "attribute_score":   float,  # 0.0 ≤ x ≤ 1.0
    "association_score": float,  # 0.0 ≤ x ≤ 1.0
}
```

This matches **`metric_interface.MetricResult`** and satisfies **`validate_metric_result`**.

## Determinism Axioms

The metric-induced preorder must satisfy:
1. **Reflexivity**:    metric(M, M) gives {class=1.0, attr=1.0, assoc=1.0}
2. **Antisymmetry**:   if q(a) == q(b), then a and b are equivalent
3. **Transitivity**:   if q(a) ≤ q(b) and q(b) ≤ q(c), then q(a) ≤ q(c)
4. **Consistency**:    all three sub-scores induce the same ranking

These are enforced by `tS-3.py`.
