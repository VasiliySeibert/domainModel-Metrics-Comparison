"text": "# Implementation Specification: Automated Class Diagram Assessment using Semantic and Structural Similarities\n\n**Adapted for PlantUML input via the existing `TestingMetrics.dummyMetric.Parser` module.**\n\n---\n\n## 1. Overview\n\nThis specification describes how to implement an automated class-diagram similarity metric that combines **semantic similarity** (lexical content) and **structural similarity** (graph topology), based on Triandini (2021).\n\nThe input format is **PlantUML** (not XMI as in the original paper). Parsing is delegated to the existing `PlantUMLParser` documented in the project's `Parser/README.md`. This specification details:\n\n1. The required dependencies.\n2. How to map the parser's data model to the paper's class-diagram concepts.\n3. The semantic similarity pipeline.\n4. The structural similarity pipeline using a UML Common Graph (UCG).\n5. The final combined metric.\n6. Adaptations made to handle parser-specific differences from the paper.\n\n---\n\n## 2. Dependencies\n\n### 2.1 Project-Internal\n\n- **`TestingMetrics.dummyMetric.Parser.PlantUMLParser`** — for converting PlantUML strings into `ParsedModel` objects.\n- **`TestingMetrics.dummyMetric.Parser.models`** — data classes: `ParsedModel`, `ParsedClass`, `ParsedAttribute`, `ParsedEnum`, `ParsedRelationship`, `RelationshipType`.\n\n### 2.2 External Libraries\n\n- **NLTK** (Python)\n  - `nltk.corpus.wordnet` — Wu-Palmer semantic similarity, lemmatization fallback\n  - `nltk.corpus.stopwords` — stopword removal\n  - (Optional) `nltk.tokenize` — tokenization fallback\n- **Stanford CoreNLP** *(or NLTK substitute)*\n  - POS Tagging with Penn Treebank tagset\n  - Lemmatization\n- **NetworkX** — graph representation\n- **NumPy** — matrix operations\n- **`networkx.algorithms.similarity`** or **`graph-tool`** — Graph Edit Distance (`graph_edit_distance` / `optimize_graph_edit_distance` for approximation)\n\n> If Stanford CoreNLP is unavailable, NLTK's `WordNetLemmatizer` and `pos_tag` are sufficient substitutes.\n\n### 2.3 WordNet Setup\n\n```python\nimport nltk\nnltk.download('wordnet')\nnltk.download('omw-1.4')\nnltk.download('stopwords')\nnltk.download('averaged_perceptron_tagger')\nnltk.download('punkt')\n```\n\n---\n\n## 3. Input: Parsed Model\n\nThe metric consumes two `ParsedModel` instances:\n\n```python\nfrom TestingMetrics.dummyMetric.Parser import PlantUMLParser\n\nparser = PlantUMLParser(strict=False)\nmodel_student = parser.parse(student_plantuml)\nmodel_key     = parser.parse(reference_plantuml)\n\nsimilarity = class_diagram_similarity(model_student, model_key,\n                                      rho=0.5, rho_sem=0.3, rho_struc=0.1)\n```\n\n### 3.1 Mapping Parser ↔ Paper Concepts\n\n| Paper concept | Parser concept | Notes |\n|---|---|---|\n| Class | `ParsedClass` | Includes `is_abstract` flag |\n| Attribute name | `ParsedAttribute.name` | |\n| Attribute type | `ParsedAttribute.type` | `None` if untyped — treat as empty string |\n| Attribute modifier | *(not directly available)* | Use `ParsedAttribute.is_constant` as proxy (\"const\" vs \"\") |\n| **Operation** | *Not supported by parser* | **See §3.2** |\n| Association (paper) | `RelationshipType.ASSOCIATION`, `DIRECTED_ASSOCIATION`, `COMPOSITION`, `AGGREGATION` | Aggregation/Composition encoded as `ownership` field |\n| Dependency (paper) | `RelationshipType.DEPENDENCY` | |\n| Generalization (paper) | `RelationshipType.INHERITANCE` | Parent = source (parser normalizes this) |\n| Relation name | `ParsedRelationship.label` | May be `None` |\n| Source class | `ParsedRelationship.source` | |\n| Target class | `ParsedRelationship.target` | |\n| Cardinalities | `source_cardinality`, `target_cardinality` | Not used in paper; **ignored** by the metric |\n| Enum | `ParsedEnum` | **See §3.3** |\n| Association Class | `RelationshipType.ASSOCIATION_CLASS` | **See §3.4** |\n\n### 3.2 Adaptation: Missing Operations\n\nThe original paper computes `cSim` (single-class similarity) as:\n\n```\ncSim = w_cn × cnSim + w_a × aSim + w_op × opSim     with (0.4, 0.3, 0.3)\n```\n\nBecause the parser does **not** parse operations, the **operation component is omitted** and weights are renormalized over the remaining components:\n\n```\ncSim = w_cn × cnSim + w_a × aSim     with (w_cn, w_a) = (0.4/0.7, 0.3/0.7)\n     = 0.571 × cnSim + 0.429 × aSim\n```\n\nSimilarly, `prSim` (parameter similarity) is removed; `relSim` is unaffected.\n\n### 3.3 Adaptation: Enums\n\nTreat each `ParsedEnum` (standalone or nested) as a class:\n- **Class name** = enum name\n- **Attributes** = list of `ParsedAttribute(name=value, type=\"enum_value\", is_constant=True)` for each enum value\n- This way enums participate in both `propSim` (class collection) and structural matching.\n\nAdd an \"enum class\" flag in the internal class representation to allow later distinction if needed; for similarity, treat them identically to classes.\n\n### 3.4 Adaptation: Association Classes\n\nFor a `RelationshipType.ASSOCIATION_CLASS` with `association_members = (X, Y)` and target = `Z`:\n- Decompose into **two associations**: `X — Z` and `Y — Z`, each with paper's `ownership = \"none\"`.\n- This preserves the structural information without requiring a new relationship category.\n\n### 3.5 Normalized Internal Representation\n\nBuild an intermediate representation `Diagram` after parsing:\n\n```python\n@dataclass\nclass Diagram:\n    classes: List[ClassInfo]            # includes enums-as-classes\n    associations: List[Edge]            # ASSOCIATION + DIRECTED_ASSOCIATION + AGGREGATION + COMPOSITION\n    dependencies: List[Edge]            # DEPENDENCY\n    generalizations: List[Edge]         # INHERITANCE\n\n@dataclass\nclass ClassInfo:\n    name: str\n    is_abstract: bool\n    attributes: List[AttributeInfo]\n    # operations omitted (parser does not provide them)\n\n@dataclass\nclass AttributeInfo:\n    name: str\n    type: str           # \"\" if untyped\n    modifier: str       # \"const\" if is_constant else \"\"\n\n@dataclass\nclass Edge:\n    source: str\n    target: str\n    name: str           # \"\" if no label\n    ownership: str      # \"composition\" | \"aggregation\" | \"none\"   (only meaningful for associations)\n```\n\n---\n\n## 4. Semantic Similarity Component\n\n### 4.1 NLP Pipeline (`CosineSim`)\n\nGiven two strings `s1`, `s2` (e.g., two class names), produce a similarity in `[0, 1]`.\n\n**Steps:**\n\n1. **Tokenization (camelCase-aware)**:\n   - Split on uppercase boundaries: `getData` → `[\"get\", \"Data\"]`\n   - Preserve uppercase runs until lowercase follows: `XMLParser` → `[\"XML\", \"Parser\"]`\n   - Also split on `_`, spaces, and digits.\n2. **Lowercase normalization** (after tokenization, before tagging).\n3. **POS tagging** with Penn Treebank tagset.\n4. **Stopword removal** using a standard English stopword list.\n5. **Lemmatization** using WordNet lemmatizer (with POS hints for accuracy).\n6. **Vector building**:\n   - Build a unified vocabulary `V` of *semantically distinct* lemmas from `s1 ∪ s2`.\n   - A lemma is added only if its Wu-Palmer similarity to all existing entries in `V` is below threshold `τ = 0.85`.\n7. **Weighted vector construction**:\n   - For each input string, build a vector of length `|V|`. Each entry is the **maximum Wu-Palmer similarity** between any token in that string and the corresponding vocabulary word.\n8. **Cosine similarity**:\n   ```\n   CosineSim(s1, s2) = (v1 · v2) / (||v1|| × ||v2||)\n   ```\n\n**Edge cases:**\n- If either string is empty → return `0`.\n- If both strings are empty → return `1`.\n- If Wu-Palmer returns `None` (word not in WordNet) → fall back to exact-string equality (`1` if equal, `0` otherwise).\n\n### 4.2 `changePivot` Helper\n\n```python\ndef change_pivot(M, x, y):\n    M[x, :] = 0\n    M[:, y] = 0\n    return M\n```\n\n### 4.3 Greedy Optimal Matching\n\n```python\ndef greedy_optimal_sum(M):\n    \"\"\"Sum of the best non-conflicting pair-matches in M.\"\"\"\n    total = 0.0\n    M = M.copy()\n    n, m = M.shape\n    for _ in range(min(n, m)):\n        x, y = np.unravel_index(np.argmax(M), M.shape)\n        total += M[x, y]\n        change_pivot(M, x, y)\n    return total\n```\n\n### 4.4 Top-Level Semantic Formula — Eq. (1)\n\n```\nclassSem(d1, d2) = (1 - ρ_sem) × propSim(d1, d2) + ρ_sem × relSim(d1, d2)\n```\n\n### 4.5 Property Similarity — Eq. (2)\n\n```\npropSim(d1, d2) = (2 × greedy_optimal_sum(cSim_matrix)) / (|C1| + |C2|)\n```\n\nWhere `C1`, `C2` are the **classes (incl. enums-as-classes)** of each diagram, and `cSim_matrix[i][j] = cSim(c_i, c_j)`.\n\nReturn `1.0` if both collections are empty; `0.0` if exactly one is empty.\n\n### 4.6 Class Similarity — Eq. (3, adapted)\n\n```\ncSim(c1, c2) = 0.571 × cnSim(c1, c2) + 0.429 × aSim(c1, c2)\n```\n\n- `cnSim = CosineSim(c1.name, c2.name)`\n- `aSim` defined below.\n\n### 4.7 Attribute Similarity — Eq. (4)\n\n```\naSim(c1, c2) = (2 × greedy_optimal_sum(daSim_matrix)) / (|A1| + |A2|)\n```\n\nIf both attribute lists are empty → `1.0`. If one is empty → `0.0`.\n\n### 4.8 Detailed Attribute Similarity — Eq. (5)\n\n```\ndaSim(a1, a2) = 0.1 × sim(a1.modifier, a2.modifier)\n              + 0.7 × CosineSim(a1.name, a2.name)\n              + 0.2 × sim(a1.type, a2.type)\n```\n\nWhere `sim(x, y) = 1.0 if x == y else 0.0` (binary equality on lowercased strings).\n\n### 4.9 Relationship Similarity — Eq. (9)\n\n```\nrelSim(d1, d2) = w_ra × raSim(d1, d2)\n               + w_rd × rdSim(d1, d2)\n               + w_rg × rgSim(d1, d2)\n```\n\n**Dynamic weights:**\n```\nN = |RA1| + |RD1| + |RG1| + |RA2| + |RD2| + |RG2|\nw_ra = (|RA1| + |RA2|) / N\nw_rd = (|RD1| + |RD2|) / N\nw_rg = (|RG1| + |RG2|) / N\n```\n\nIf `N == 0` → return `1.0` if both diagrams truly have no relationships; otherwise `0.0`.\n\n### 4.10 Association Similarity — Eqs. (10–11)\n\n```\nraSim(d1, d2) = (2 × greedy_optimal_sum(draSim_matrix)) / (|RA1| + |RA2|)\n\ndraSim(ra1, ra2) = 0.3 × CosineSim(ra1.source, ra2.source)\n                 + 0.2 × CosineSim(ra1.name,   ra2.name)\n                 + 0.2 × sim(ra1.ownership,   ra2.ownership)\n                 + 0.3 × CosineSim(ra1.target, ra2.target)\n```\n\nIf either relationship name is empty/`None`, treat the name term as `0` (paper does not skip; the empty cosine returns 0).\n\n### 4.11 Dependency & Generalization Similarity — Eqs. (12–15)\n\n```\nrdSim(d1, d2) = (2 × greedy_optimal_sum(drdSim_matrix)) / (|RD1| + |RD2|)\ndrdSim(rd1, rd2) = (CosineSim(src1, src2) + CosineSim(tgt1, tgt2)) / 2\n\nrgSim(d1, d2) = (2 × greedy_optimal_sum(drgSim_matrix)) / (|RG1| + |RG2|)\ndrgSim(rg1, rg2) = (CosineSim(src1, src2) + CosineSim(tgt1, tgt2)) / 2\n```\n\n---\n\n## 5. Structural Similarity Component\n\n### 5.1 UML Common Graph (UCG) Construction\n\n**Adapted vertex/edge taxonomy** (operation/parameter vertices and edges are dropped since the parser doesn't expose operations):\n\n| Type | Name | Tag |\n|------|------|-----|\n| Vertex | Class Vertex | `vc` |\n| Vertex | Attribute Vertex | `va` |\n| Edge | Attribute Edge | `e_a` |\n| Edge | Association Edge | `e_1` |\n| Edge | Generalization Edge | `e_2` |\n| Edge | Aggregation Edge | `e_3` |\n| Edge | Composition Edge | `e_4` |\n| Edge | Dependency Edge | `e_5` |\n\n**Construction (`build_ucg(diagram) -> nx.MultiDiGraph`):**\n1. For each class in `diagram.classes`, add a `vc` vertex (label `vc`).\n2. For each attribute of that class, add a `va` vertex and connect to the class vertex via an `e_a` edge.\n3. For each relationship:\n   - Association with `ownership == \"none\"` → `e_1`\n   - Association with `ownership == \"aggregation\"` → `e_3`\n   - Association with `ownership == \"composition\"` → `e_4`\n   - Generalization → `e_2`\n   - Dependency → `e_5`\n\n**Important:** Vertices/edges retain only their **type tag**, *not* lexical labels. Nodes can be matched by tag identity only.\n\n### 5.2 Top-Level Structural Formula — Eq. (16)\n\n```\nclassStruc(d1, d2) = (1 - ρ_struc) × intraSim(g1, g2) + ρ_struc × interSim(g1, g2)\n```\n\n### 5.3 Intra-structure Similarity — Eq. (17)\n\n**Partition** each UCG into per-class subgraphs:\n- `intra_subgraph(c)` = the class vertex `vc` plus all its incident attribute vertices `va` and `e_a` edges.\n\n**Compute:**\n```python\ndef intra_sim(g1, g2):\n    sg1_list = intra_subgraphs(g1)\n    sg2_list = intra_subgraphs(g2)\n    if not sg1_list and not sg2_list: return 1.0\n    if not sg1_list or not sg2_list:  return 0.0\n\n    M = np.zeros((len(sg1_list), len(sg2_list)))\n    for i, sg1 in enumerate(sg1_list):\n        for j, sg2 in enumerate(sg2_list):\n            M[i, j] = ged_normalized(sg1, sg2)\n\n    return 2 * greedy_optimal_sum(M) / (len(sg1_list) + len(sg2_list))\n```\n\n### 5.4 Normalized Graph Edit Distance — Eq. (18)\n\n```python\ndef ged_normalized(sg1, sg2):\n    cost = approx_ged(sg1, sg2)        # see §5.6\n    size = max(len(sg1.nodes) + len(sg1.edges),\n               len(sg2.nodes) + len(sg2.edges))\n    if size == 0:\n        return 1.0\n    return 1 - (cost / size)\n```\n\n**Edit operations** (all unit cost = 1):\n- **Substitution** of vertex/edge type\n- **Insertion** of vertex/edge\n- **Deletion** of vertex/edge\n\nTwo vertices/edges are considered equal iff their **type tag** matches.\n\n### 5.5 Inter-structure Similarity — Eq. (19)\n\n**Extract** the inter graph: only `vc` vertices and the relationship edges (`e_1` … `e_5`) between them.\n\n```python\ndef inter_sim(g1, g2):\n    ig1 = inter_graph(g1)\n    ig2 = inter_graph(g2)\n    cost = approx_ged(ig1, ig2)\n    size = max(len(ig1.nodes) + len(ig1.edges),\n               len(ig2.nodes) + len(ig2.edges))\n    if size == 0:\n        return 1.0\n    return 1 - (cost / size)\n```\n\n### 5.6 GED Implementation\n\nFor small graphs (≲ 15 nodes), use exact computation:\n```python\ncost = nx.graph_edit_distance(\n    g1, g2,\n    node_match=lambda a, b: a['tag'] == b['tag'],\n    edge_match=lambda a, b: a['tag'] == b['tag'],\n)\n```\n\nFor larger graphs, use approximation:\n```python\ngen = nx.optimize_graph_edit_distance(g1, g2, node_match=..., edge_match=...)\ncost = min(itertools.islice(gen, 100))   # take best of first 100 candidates\n```\n\nOr replace with the Riesen–Bunke bipartite-matching approximation if performance is critical.\n\n---\n\n## 6. Combined Similarity — Eq. (20)\n\n```python\ndef class_diagram_similarity(model1, model2, rho=0.5, rho_sem=0.3, rho_struc=0.1):\n    d1 = normalize(model1)      # ParsedModel -> Diagram (§3.5)\n    d2 = normalize(model2)\n\n    sem  = class_sem(d1, d2, rho_sem)\n    struc = class_struc(d1, d2, rho_struc)\n\n    return (1 - rho) * sem + rho * struc\n```\n\n### Recommended Parameter Defaults (from paper's max-agreement experiment)\n\n| Parameter | Value | Meaning |\n|---|---|---|\n| `rho`        | `0.5` | Equal weight to semantic and structural |\n| `rho_sem`    | `0.3` | Properties weighted more than relationships (within semantic) |\n| `rho_struc`  | `0.1` | Intra weighted more than inter (within structural) |\n\nThe paper achieved Gwet's AC1 = 0.935 (almost perfect agreement with human experts) at these values.\n\n---\n\n## 7. End-to-End Pseudocode\n\n```python\ndef normalize(parsed: ParsedModel) -> Diagram:\n    classes = []\n\n    # 1. Regular classes\n    for cls in parsed.classes:\n        attrs = [\n            AttributeInfo(name=a.name,\n                          type=(a.type or \"\").lower(),\n                          modifier=\"const\" if a.is_constant else \"\")\n            for a in cls.attributes\n        ]\n        classes.append(ClassInfo(cls.name, cls.is_abstract, attrs))\n\n        # Nested enums become classes too\n        for enum in cls.nested_enums:\n            classes.append(_enum_to_class(enum))\n\n    # 2. Top-level enums\n    for enum in parsed.enums:\n        classes.append(_enum_to_class(enum))\n\n    # 3. Implicit classes (mentioned only in relationships)\n    for name in parsed.implicit_classes:\n        classes.append(ClassInfo(name, False, []))\n\n    # 4. Relationships\n    associations, dependencies, generalizations = [], [], []\n    for rel in parsed.relationships:\n        edges = _to_edges(rel)            # handles ASSOCIATION_CLASS expansion\n        for e, kind in edges:\n            if   kind == \"association\":   associations.append(e)\n            elif kind == \"dependency\":    dependencies.append(e)\n            elif kind == \"generalization\": generalizations.append(e)\n\n    return Diagram(classes, associations, dependencies, generalizations)\n\n\ndef _enum_to_class(enum: ParsedEnum) -> ClassInfo:\n    return ClassInfo(\n        name=enum.name,\n        is_abstract=False,\n        attributes=[AttributeInfo(v, \"enum_value\", \"const\") for v in enum.values],\n    )\n\n\ndef _to_edges(rel: ParsedRelationship):\n    t = rel.relationship_type\n    name = rel.label or \"\"\n    if t == RelationshipType.INHERITANCE:\n        return [(Edge(rel.source, rel.target, name, \"none\"), \"generalization\")]\n    if t == RelationshipType.DEPENDENCY:\n        return [(Edge(rel.source, rel.target, name, \"none\"), \"dependency\")]\n    if t == RelationshipType.COMPOSITION:\n        return [(Edge(rel.source, rel.target, name, \"composition\"), \"association\")]\n    if t == RelationshipType.AGGREGATION:\n        return [(Edge(rel.source, rel.target, name, \"aggregation\"), \"association\")]\n    if t in (RelationshipType.ASSOCIATION, RelationshipType.DIRECTED_ASSOCIATION):\n        return [(Edge(rel.source, rel.target, name, \"none\"), \"association\")]\n    if t == RelationshipType.ASSOCIATION_CLASS:\n        x, y = rel.association_members\n        z = rel.target\n        return [\n            (Edge(x, z, name, \"none\"), \"association\"),\n            (Edge(y, z, name, \"none\"), \"association\"),\n        ]\n    return []\n```\n\n---\n\n## 8. Output\n\nThe metric returns a float in `[0, 1]`.\n\nFor grading / display:\n- Scale to `[0, 100]` (multiply by 100).\n- Optionally map to a 1–5 ordinal scale matching the paper:\n  - 1: `< 0.20`, 2: `< 0.40`, 3: `< 0.60`, 4: `< 0.80`, 5: `≥ 0.80`.\n\n---\n\n## 9. Edge Cases & Notes\n\n1. **Empty diagrams**: Treat similarity of two empty collections as `1.0`; one empty and one non-empty as `0.0`.\n2. **Missing types/modifiers**: Use empty strings for `None`.\n3. **Words missing from WordNet**: Wu-Palmer falls back to exact-string equality.\n4. **Unlabeled relationships**: `CosineSim(\"\", \"\") = 1.0` (per paper convention). If you prefer to ignore unlabeled name comparisons, redistribute the `0.2` weight from `w_ra2` to source/target equally.\n5. **Implicit classes** (from `model.implicit_classes`): include them as zero-attribute classes so structural counts remain consistent.\n6. **Abstract classes**: `is_abstract` is *not* used by the metric (paper does not consider it). Optionally extend `cSim` to add a small abstract-equality term if needed.\n7. **Large diagrams**: Use approximate GED (`nx.optimize_graph_edit_distance`) to avoid combinatorial blow-up.\n8. **Ownership encoding**: Inside `draSim`, `ownership` ∈ {\"none\", \"aggregation\", \"composition\"} — comparison is binary equality.\n\n---\n\n## 10. Summary of Adaptations vs. Original Paper\n\n| Aspect | Paper | This Implementation |\n|---|---|---|\n| Input format | XMI | **PlantUML** (via `PlantUMLParser`) |\n| Operations | Included with `w_op = 0.3` | **Removed**; class weights renormalized to `(0.571, 0.429)` |\n| Parameters (`prSim`) | Included | **Removed** (no operations) |\n| Modifiers | Public/private/etc. | Approximated via `is_constant` |\n| Enums | Not mentioned | Treated as **classes with `enum_value` attributes** |\n| Association Class | Not mentioned | **Expanded into two associations** |\n| UCG vertices | vc, va, vo, vp, vob | **vc, va** only (no vo, vp) |\n| UCG edges | 13 types | **6 types** (e_a, e_1, e_2, e_3, e_4, e_5) |\n| Cardinalities | Not used | Not used (consistent with paper) |\n| All other formulas | As in paper | Unchanged |\n\n---\n\n## 11.
