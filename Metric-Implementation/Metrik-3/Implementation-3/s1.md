# S1 Specification -- Structural Similarity of UML Class Diagrams via UCG

## Overview

The structural similarity metric compares two parsed PlantUML class diagrams
by converting each into a **UML Class Graph (UCG)**, then measuring how much
of their inter-structure (relationship topology) and intra-structure
(attribute/operation composition) overlaps.

The metric returns a single float in the range **[0.0, 1.0]**, where **0.0**
means completely different and **1.0** means identical.

```python
metric(instructor_model, student_model):
    instructor_ucg = transformUCDtoUCG(instructor_model)
    student_ucg    = transformUCDtoUCG(student_model)
    umcs_tree      = buildUMCSTree(instructor_ucg, student_ucg)
    inter_candidates = computeInterStructureSimilarity(instructor_ucg, student_ucg, umcs_tree)
    similarities = []
    for sim_inter, matching_pairs in inter_candidates:
        sim_intra = computeIntraStructureSimilarity(instructor_ucg, student_ucg, matching_pairs)
        sim       = combineStructuralSimilarity(sim_inter, sim_intra)  # THETA = 0.9
        similarities.append(sim)
    similarity = max(similarities) if similarities else 0.0
    return similarity

    requires:
        isValidModel(instructor_model),
        isValidModel(student_model)
    ensures:
        isValidSimilarity(similarity)
```

## Invariants

### isValidModel(M)

```
isValidModel(M):
    checks if model adheres to the ParsedModel data structure.
    class names are unique within M
    enumeration names are unique within M (standalone + nested)
    every relation references classes that exist in M
    every attribute belongs to exactly one class in M
```

### isValidUCG(G)

```
isValidUCG(G):
    G is an UML Class Graph G = (V, E, L) where
    V = CV U AV U OV U PV
    E = AE U OE U PE U RE
    CV  -- class vertices       (vertex_type: "class",     tag concept: "v0")
    AV  -- attribute vertices   (vertex_type: "attribute", tag concept: "v1")
    OV  -- operation vertices   (vertex_type: "operation", tag concept: "v2")
    PV  -- parameter vertices  (vertex_type: "parameter", tag concept: "v3")
    AE  -- attribute edges      (edge_type: "attribute",    tag: "ea")
    OE  -- operation edges     (edge_type: "operation",    tag: "eo")
    PE  -- parameter edges     (edge_type: "parameter",    tag: "ep")
    RE  -- relationship edges  (edge_type: "relationship", tags: "e1".."e6")

    1. G is a UCG instance
    2. Every vertex is a UCGVertex with a valid vertex_type
    3. Every edge is a UCGEdge with a valid edge_type and tag
    4. Vertex IDs are unique within V
    5. Edge IDs are unique within E
    6. Every edge references vertices that exist in G
    7. Endpoint types match the edge type constraints:
       * attribute edges:   class → attribute
       * operation edges:   class → operation
       * parameter edges:   operation → parameter
       * relationship edges: class → class
    8. All vertex labels are non-empty strings
    9. No non-class vertex is orphaned (must have at least one incident edge)
    10. Tree-shape constraints:
        * attribute vertex: exactly 1 incoming attribute edge, 0 outgoing edges
        * operation vertex: exactly 1 incoming operation edge, 0 outgoing edges
        * parameter vertex: exactly 1 incoming parameter edge, 0 outgoing edges
    11. No duplicate edges (same source, target, edge_type, tag)
    12. Tag must be consistent with edge_type (e.g. attribute edges carry tag "ea")
```

### isValidUMCSTree(T)

```
isValidUMCSTree(T):
    T is a UMCSNode representing a UMCS Tree.

    1. Root node is a UMCSNode instance
    2. At each node, every entry in mcsl is a UMCS instance
    3. At each node, all UMCS in mcsl have the same cardinality
    4. All UMCS in a node's mcsl are pairwise distinct
    5. Along any root-to-leaf path, UMCS cardinalities are non-increasing
```

### isValidSimilarity(s)

```
isValidSimilarity(s):
    s is an instance of int or float
    s is finite (not NaN, not +inf, not -inf)
    0.0 <= s <= 1.0

    Range semantics:
      0.0 -- completely different
      1.0 -- structurally identical
      (0.0, 1.0) -- degrees of partial structural overlap
```

## Decompose functions

### transformUCDtoUCG

```
ucg = transformUCDtoUCG(model):
    V = createVerticesForElements(model.classes, model.enums, model.implicit_classes)
    E = createEdgesForAttributes(model.classes)
      + createEdgesForRelations(model.relationships)
    L = assignLabels(V)
    return G = (V, E, L)

    requires:
        isValidModel(model)
    ensures:
        isValidUCG(G)

    Input:  A ParsedModel adhering to isValidModel.
    Output: A UML Class Graph (UCG) instance adhering to isValidUCG.

    Transformation rules:
      Rule 1: class ci -> class vertex cvi, label = ci.name, vertex_type = "class"
      Rule 2: attribute aij -> attribute vertex avij (vertex_type "attribute")
               + attribute edge aeij (tag "ea") from cvi to avij
      Rule 3: OMITTED -- The parser does not extract operations or parameters,
               so operation vertices, operation edges, parameter vertices and
               parameter edges are not produced by this transformation.
      Rule 4: relationship (ci, tx, cj) -> relationship edge reij (tag ex)
               from cvi to cvj, where ex is mapped from tx:

                 +-----------------+----------------------+
                 | Relationship tx | Tag ex               |
                 +-----------------+----------------------+
                 | association     | e1                   |
                 | directed        | e1                   |
                 | inheritance     | e2                   |
                 | aggregation     | e3                   |
                 | composition     | e4                   |
                 | dependency      | e5                   |
                 +-----------------+----------------------+

    Direction convention (source -> target):
      * association          : source = first class in line, target = second class
      * directed association : source = tail of arrow, target = head of arrow
      * inheritance          : source = parent, target = child
      * composition          : source = composite/whole, target = part
      * aggregation          : source = whole, target = part
      * dependency           : source = dependent, target = target
      * association_class    : ignored (no UCG representation defined)

    ID generation convention (deterministic):
      class vertex  -> "cv:{class_name}"
      attribute vertex -> "av:{class_name}:{attr_name}"
      attribute edge   -> "ae:{class_name}:{attr_name}"
      relationship edge -> "re:{source}:{target}:{tag}"
```

### buildUMCSTree

```
umcs_tree = buildUMCSTree(g1, g2):
    root = new UMCSNode()
    mcsl = computeUMCSList(g1, g2)
    if mcsl is not empty:
        root.mcsl = mcsl
        for each umcs in mcsl (in order returned by computeUMCSList):
            g1' = g1 without relationship edges whose edge_id is in umcs.edge_ids
            g2' = g2 without the corresponding matched relationship edges
                  (identified via umcs.vertex_map and edge tag equality)
            child = buildUMCSTree(g1', g2')
            append child to root.children
    return root

    requires:
        isValidUCG(g1),
        isValidUCG(g2)
    ensures:
        isValidUMCSTree(umcs_tree)

    Input:  Two valid UCG instances.
    Output: The root node of a UMCS Tree.

    Subtraction semantics:
    - Only relationship edges are removed.
    - Class vertices, attribute vertices, attribute edges, and all other
      non-relationship elements remain, even if a class vertex ends up with
      zero remaining relationship edges.
    - The g2 edges to remove are identified by translating g1 edge endpoints
      through umcs.vertex_map and matching on (mapped_source, mapped_target, tag).

    Termination proof:
    Each recursion removes at least one relationship edge from both g1 and g2.
    The recursion depth is bounded by min(|RE_g1|, |RE_g2|) + 1.
```

### computeUMCSList

```
mcsl = computeUMCSList(g1, g2):
    mcsl = empty list
    S = empty stack of (g1_edge_id, g2_edge_id) pairs
    vertex_map = empty dict   # g1_class_id -> g2_class_id
    inv_map    = empty dict   # g2_class_id -> g1_class_id
    used_g2    = empty set    # g2 edge ids already consumed
    best_size  = 0

    E1 = sorted(g1.relationship_edges(), key=(tag, source_id, target_id))
    E2 = sorted(g2.relationship_edges(), key=(tag, source_id, target_id))

    backtrack(i):
        if i == len(E1):
            if len(S) > best_size:
                best_size = len(S)
                mcsl = [UMCS(edge_ids=frozenset({e1 for e1, _ in S}),
                             vertex_map=dict(vertex_map))]
            elif len(S) == best_size and len(S) > 0:
                candidate = UMCS(edge_ids=frozenset({e1 for e1, _ in S}),
                                   vertex_map=dict(vertex_map))
                if candidate not in mcsl:
                    mcsl.append(candidate)
            return

        # Branch A: skip E1[i]
        backtrack(i + 1)

        # Branch B: try to include E1[i]
        for each e2 in E2 with e2.tag == E1[i].tag and e2.edge_id not in used_g2:
            s1, t1 = E1[i].source, E1[i].target
            s2, t2 = e2.source, e2.target

            # Consistency check
            feasible = True
            if vertex_map.get(s1) not in (None, s2): feasible = False
            if vertex_map.get(t1) not in (None, t2): feasible = False
            if inv_map.get(s2)    not in (None, s1): feasible = False
            if inv_map.get(t2)    not in (None, t1): feasible = False

            if feasible:
                # Push state
                added_mappings = []
                if s1 not in vertex_map: added_mappings.append((s1, s2))
                if t1 not in vertex_map: added_mappings.append((t1, t2))
                for (a, b) in added_mappings:
                    vertex_map[a] = b
                    inv_map[b] = a
                S.append((E1[i].edge_id, e2.edge_id))
                used_g2.add(e2.edge_id)

                backtrack(i + 1)

                # Pop state
                S.pop()
                used_g2.remove(e2.edge_id)
                for (a, b) in added_mappings:
                    del vertex_map[a]
                    del inv_map[b]

    backtrack(0)
    return mcsl

    requires:
        isValidUCG(g1),
        isValidUCG(g2)
    ensures:
        Every UMCS in mcsl is a UMCS instance with:
          - edge_ids: frozenset of valid relationship edge IDs from g1
          - vertex_map: Dict[g1_class_id, g2_class_id], injective and consistent
        All UMCS in mcsl have the same cardinality (maximum possible).
        All UMCS in mcsl are pairwise distinct.

    Input:  Two valid UCG instances.
    Output: A list of UMCS (UML Maximum Common Subgraph) objects.

    Algorithm:
    Exact exponential-time backtracking.  E1 and E2 are sorted deterministically
    by (tag, source_id, target_id).  The algorithm explores every subset of E1
    that can be matched injectively to a subset of E2 with the same tags,
    maintaining a consistent vertex mapping.  The maximum-size matches are
    collected in mcsl.

    Note: The exponential complexity is acceptable in practice because UCGs
    produced by this metric typically contain no more than 30 relationship
    edges.
```

### computeInterStructureSimilarity

```
inter_candidates = computeInterStructureSimilarity(g1, g2, umcs_tree):
    candidates = []
    for each UMCSS path (pre-order traversal of umcs_tree):
        total_size = sum(len(umcs.edge_ids) for umcs in UMCSS)
        sim_inter = total_size / min(|RE_g1|, |RE_g2|)
        matching_pairs = empty set
        for each umcs in UMCSS:
            for each edge_id in umcs.edge_ids:
                e1 = edge in g1 with edge_id
                s1 = e1.source_vertex_id
                t1 = e1.target_vertex_id
                s2 = umcs.vertex_map[s1]
                t2 = umcs.vertex_map[t1]
                matching_pairs.add((s1, s2))
                matching_pairs.add((t1, t2))
        candidates.append( (sim_inter, matching_pairs) )
    return candidates

    requires:
        isValidUCG(g1),
        isValidUCG(g2),
        isValidUMCSTree(umcs_tree),
        |RE_g1| > 0,
        |RE_g2| > 0
    ensures:
        Every sim_inter is in [0, 1].
        Every pair in matching_pairs references class vertices that exist in
        g1 and g2 respectively.
        matching_pairs contains no duplicate pairs.
        The returned list may be empty (when no common subgraph exists).

    Input:  Two valid UCGs (each with at least one relationship edge) and a
            valid UMCS Tree.
    Output: List of (sim_inter, matching_pairs) candidate tuples.
            matching_pairs is a Set[Tuple[str, str]].

    Algorithm:
    Enumerates all UMCSS paths via pre-order traversal, computes total size,
    and normalises by the minimum number of relationship edges in either graph.
    Matching pairs are derived from umcs.vertex_map without recomputing the
    correspondence.

    Formula:
      sim_inter = |UMCSS| / min(|RE_g1|, |RE_g2|)
      where |UMCSS| = sum of |UMCS.edge_ids| for each UMCS in the path.
```

### computeIntraStructureSimilarity

```
sim_intra = computeIntraStructureSimilarity(g1, g2, matching_pairs):
    for each (cvi, cvj) in matching_pairs:
        AVi = attribute vertices attached to cvi via "ea" edges
        AVj = attribute vertices attached to cvj via "ea" edges
        na_i = |AVi|, na_j = |AVj|
        x1 = max(0, na_j - na_i)   -- insertions needed
        y1 = max(0, na_i - na_j)   -- deletions needed

        OVi = operation vertices attached to cvi via "eo" edges
        OVj = operation vertices attached to cvj via "eo" edges
        no_i = |OVi|, no_j = |OVj|
        x2 = max(0, no_j - no_i)
        y2 = max(0, no_i - no_j)

        for each matched operation pair (ovi_k, ovj_w):
            PVik = parameter vertices attached to ovi_k via "ep" edges
            PVjw = parameter vertices attached to ovj_w via "ep" edges
            np_ik = |PVik|, np_jw = |PVjw|
            x3_k = max(0, np_jw - np_ik)
            y3_k = max(0, np_ik - np_jw)

    simAttr  = 1 - sum(x1+y1)*IC1 / sum(max(na_i, na_j))
    simOper  = 1 - sum(x2+y2)*IC2 / sum(max(no_i, no_j))
    simParam = 1 - sum(x3+y3)*IC3 / sum(max(np_ik, np_jw))
    sim_intra = ALPHA * simAttr + BETA * simOper + GAMMA * simParam
    return sim_intra

    ALPHA = 0.4, BETA = 0.5, GAMMA = 0.1
    IC1 = IC2 = IC3 = 1

    requires:
        isValidUCG(g1),
        isValidUCG(g2)
    ensures:
        isValidSimilarity(sim_intra)
        result == ALPHA * simAttr + BETA * simOper + GAMMA * simParam

    Input:  Two valid UCGs and a set of matched class-vertex pairs.
    Output: Intra-structure similarity value in [0, 1].

    Algorithm:
    Uses UCG edit distance on attribute, operation, and parameter
    vertices.  The edit distance counts insertions and deletions only;
    vertex labels are ignored because structural similarity is based on
    topology, not semantics.

    Scope limitation (concrete parser):
    Because the concrete PlantUML parser does not extract operations or
    parameters, the UCGs never contain operation vertices, operation edges,
    parameter vertices, or parameter edges.  Consequently:
        simOper always evaluates to 1.0 (zero edit cost)
        simParam always evaluates to 1.0 (zero edit cost)
    The effective formula reduces to:
        sim_intra = 0.4 * simAttr + 0.5 + 0.1 = 0.4 * simAttr + 0.6
```

### combineStructuralSimilarity

```
similarity = combineStructuralSimilarity(sim_inter, sim_intra):
    return THETA * sim_inter + (1 - THETA) * sim_intra

    THETA = 0.9  (module-level constant, fixed)

    requires:
        isValidSimilarity(sim_inter),
        isValidSimilarity(sim_intra)
    ensures:
        isValidSimilarity(similarity)
        result == THETA * sim_inter + (1 - THETA) * sim_intra

    Input:  Two similarity values in [0, 1].
    Output: Combined structural similarity in [0, 1].

    Rationale:
      Inter-structure (relationship topology) is experimentally found to be
      the dominant driver of perceived structural similarity, while
      intra-structure (attribute composition) provides a secondary refinement.
      Therefore THETA is fixed at 0.9.
```
