# Results

Output of running each metric over the dataset. Generated by
`Workflows/run_metrikN.py` and `compare_metrics.py`.

## Files

```
Results/
├── README.md                          # this file
├── results_metrik1.json               # Metrik-1 (rule-based)
├── results_metrik2.json               # Metrik-2 (graph edit distance)
├── results_metrik3.json               # Metrik-3 (UCG)
├── results_metrik4.json               # Metrik-4 (S-1 v1)
├── results_metrik5.json               # Metrik-5 (S-1 v2)
├── metrics_comparison.json            # cross-metric summary (MAD table)
├── metrics_comparison_histogram.png   # cross-metric MAD histogram
└── .baseline/                         # preserved old result files (for verification)
    ├── results_dmetrik4.json
    ├── results_dmetrik5.json
    ├── results_dmetrik8.json
    ├── results_dmetrik91.json
    └── results_dmetrik92.json
```

## `metrics_comparison_histogram.png`

A small-multiples bar chart of per-metric, per-element MAD histograms.
Generated by `compare_metrics.py`. Six buckets: `<0.10`, `0.10-0.15`,
`0.15-0.20`, `0.20-0.25`, `0.25-0.30`, `>=0.30`. Green = close (good), red =
far (bad). A dashed reference line at 0.20 marks the "close enough"
threshold.

## Reproducibility

The 5 result files in this directory are **bit-exact identical** to the 5
old baseline files preserved in `.baseline/`. The mapping is:

| New                  | Old (in `.baseline/`)     |
|----------------------|---------------------------|
| `results_metrik1.json` | `results_dmetrik4.json`  |
| `results_metrik2.json` | `results_dmetrik5.json`  |
| `results_metrik3.json` | `results_dmetrik8.json`  |
| `results_metrik4.json` | `results_dmetrik91.json` |
| `results_metrik5.json` | `results_dmetrik92.json` |

This was verified during the project restructuring (see commit history).
The discarded metric results are not preserved — they had the worst overall
MAD (0.277 and 0.303) and were dropped from the restructured project.
