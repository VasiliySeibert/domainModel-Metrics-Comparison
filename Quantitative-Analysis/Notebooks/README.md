# Notebooks

Jupyter walkthroughs for the project.

## `comparisons_walkthrough.ipynb`

A guided tour of the 5 metrics on a single example pair (`LabTracker` /
`0shot`), followed by a cross-metric comparison using the pre-computed
`metrics_comparison.json`.

### Structure

1. **Setup** — resolve paths, import helpers.
2. **Dataset** — load `Dataset/combined-data.json`, show the 8 × 5 grid.
3. **Example pair** — pick `LabTracker_0shot`, show the reference and
   generated PlantUML and the human F1.
4. **5 metric scores (pre-computed)** — load `Results/results_metrikN.json`
   and present the 3-score table per metric.
5. **Per-element delta** — `metric_score − human_f1` per (metric, element).
6. **Scatter plot** — metric score vs human F1, with the `y = x` reference
   line.
7. **Cross-metric comparison** — load `metrics_comparison.json`, sort by
   overall MAD, identify the best metric per element.
8. **Histogram** — embed the `metrics_comparison_histogram.png` generated by
   `compare_metrics.py`.
9. **Live re-run (gated)** — a cell with `LIVE = False` that, when flipped
   to `True`, re-runs all 5 metrics on the example pair from scratch
   (~2 minutes).
10. **Reproduce** — the `RunMetrics/run_all.py` + `compare_metrics.py`
    commands needed to regenerate everything from scratch.
11. **For the thesis** — pointers to specs, results, and discussion.

### Running

```bash
# (1) Install the notebooks extra
pip install -e ".[notebooks]"

# (2) Make sure Results/ contains the pre-computed results
python3 Quantitative-Analysis/RunMetrics/run_all.py    # ~40 min
python3 compare_metrics.py

# (3) Open the notebook
jupyter lab Quantitative-Analysis/Notebooks/comparisons_walkthrough.ipynb
```

Or to execute it headless:

```bash
jupyter nbconvert --to notebook --execute --inplace \
  Quantitative-Analysis/Notebooks/comparisons_walkthrough.ipynb
```

### Editing / regenerating

The notebook is generated by `_build_walkthrough.py` in this directory. If
you want to add or change cells, edit that script and re-run it. This keeps
the notebook content in version-controllable text form (the `.ipynb` JSON
itself is a build artefact).
